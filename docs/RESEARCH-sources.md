# NBA dynasty source landscape — research notes

Living document. New entries land as we evaluate new sources. Format
borrowed from Dynasty-Football-Model's research doc.

For each source: what it is, why it matters, accessibility (API / ToS),
expected value-add to the composite, and whether it lands in PR #1 or
later.

---

## 1. DARKO — Daily Adjusted Regressed Kalman Optimized

**Site:** https://apanalytics.shinyapps.io/DARKO/
**Author:** Kostya Medvedovsky.
**Status in PR #1:** LIVE.

The only free, public, transparent NBA player-impact metric with a
built-in longevity model. Two outputs we care about:

1. **DPM (Defensive Plus Minus style)** — current-season impact metric
   broken into O-DPM, D-DPM, Box DPM. Updates daily during the season.
   Includes a "DPM Improvement" column (vs. prior season) which is a
   strong signal for young-player risers.
2. **Survival / longevity model** — for every player, estimated
   retirement age and per-year probability of still playing for the
   next 12 years. Exactly the input a dynasty model needs.

**Access:** Shiny app, no JSON API. Scraped via the WebSocket protocol
the page itself uses. Protocol details in `sources/darko.py` module
docstring. Outage risk is mitigated by a CSV fallback at
`data/darko/darko_dump.csv`.

**Default weight:** 1.5 (highest). Justified because DARKO is the only
source giving us *both* impact AND longevity. Once we have a Production
backtest, this weight floats with realized correlation.

## 2. Court Consensus

**Site:** https://courtconsensus.com/rankings
**Status in PR #2:** LIVE.

Crowdsourced dynasty NBA rankings derived from head-to-head community
voting (76k+ data points and growing). Each player carries two ELO
scores: `elo_rating_points` (points-league) and
`elo_rating_categories` (cat-league). The site is *not* an aggregator
of other experts — it's primary consensus data, generated by their
own voter pool. That makes it the NBA analog of FantasyCalc on the
football side, and gives it a unique role next to DARKO (DARKO is a
model; CC is the market).

**Access:** Vite/React SPA backed by a public Supabase project at
`viayfcilcpoprzucmztv.supabase.co`. The anon JWT is publicly embedded
in the site's JS bundle and gated by RLS (reads are public; vote
endpoints are rate-limited per ToS §5). The adapter hits
`/rest/v1/players` once per launcher run and emits one record per
filtered player (PICK rows excluded). CSV fallback at
`data/court_consensus/court_consensus_dump.csv`.

**Default weight:** 1.0 — standard consensus baseline. DARKO stays
at 1.5 because it carries an impact metric AND a longevity signal
that CC by design does not. Once we have a Production loader and a
backtest, both weights float on track-record multiplier.

**Filtered out:** All `position == "PICK"` rows and any name matching
the `"<year> Pick / Early / Mid / Late <slot>"` pattern. CC includes
~39 such picks today; we keep only real NBA players (~303 rows).

**Why it matters for PR #2:** Sands down DARKO's rookie inflation.
DARKO ranked Knueppel/Queen/Clingan in the top 15 because the
longevity bonus compounded; CC's community ranks those same players
#34/#37/#43 respectively. Composite settles in the middle and the
`rank_divergence` field surfaces the disagreement directly.

**Attribution:** Rankings copyright Court Consensus. The site is
linked from `/sources.html`.

## 3. Sam Vecenie (The Athletic)

**Site:** https://theathletic.com/author/sam-vecenie/
**Status in PR #2:** LIVE (CSV-only — yields nothing until file dropped).

The Athletic's lead NBA Draft analyst and the de-facto industry
consensus for pre-NBA prospect evaluation. Football analog: Lance
Zierlein. The Big Board itself is paywalled at The Athletic so we
cannot scrape it; instead the adapter reads a manual CSV drop at
`data/vecenie/vecenie_big_board.csv` — same pattern as the football
repo's RAS and CFBD-Breakouts adapters.

**Schema:** see `data/vecenie/README.md`. Required columns: `rank`,
`player_name`. Optional: `position`, `tier`, `draft_year`, `notes`.
Header names are case-insensitive with sensible aliases.

**Default weight:** 1.3 — elevated single-analyst weight (matches
what we'd give Lance Zierlein on the football side). Track-record
multiplier will adjust once we have a production backtest.

**Filter behavior:** Registered in
`weights.ROOKIE_SIGNAL_SOURCES`. Players whose ONLY ranking comes
from Vecenie are filtered out of the top of the composite to prevent
hyped pre-NBA prospects from squatting on top spots before they've
played. Once a player makes their first NBA appearance, DARKO and
Court Consensus pick them up automatically and the filter releases.

**Value mapping:** Rank 1 → 100, max-rank → 0 (linear taper). Lands
in the same 0–100 band as Court Consensus + DARKO so the value-based
scoring path picks it up at the right magnitude.

**Status today:** No CSV dropped yet. Adapter yields 0 rows, launcher
continues. When Phil drops a Big Board CSV, it engages automatically
on the next refresh — no code changes needed.

## 4. Hashtag Basketball — Dynasty Rankings

**Site:** https://hashtagbasketball.com/
**Status in PR #1:** Not loaded.

One of the largest free fantasy-basketball ranking sites. Publishes
per-format dynasty rankings (9-cat, 8-cat, points-league). Scraping
their public ranking pages is feasible.

Anticipated weight: 1.0 (consensus aggregator). Will exercise the
production-based scoring branch — their per-format dynasty rankings
ARE format-dependent so the same `RankingRecord` cannot be re-used
across formats.

**Status in PR #1:** Not loaded.

FiveThirtyEight's RAPTOR metric — frozen in 2023 after the site's

## Source taxonomy (for the weighting model)

Same four categories as the football repo:

- **market** — crowdsourced trade values (FantasyCalc-equivalent for NBA
  is sparser; KTC has an NBA spinoff but is largely closed).
- **aggregator** — sites that average multiple experts (Court
  Consensus, FantasyPros NBA).
- **expert** — single-analyst rankings (Sam Vecenie, Lance Stephenson).
- **model** — algorithmic systems (DARKO, RAPTOR-historical, eventual
  Production-derived models).

Rookie-signal sources (data is pre-NBA) get added to
`weights.ROOKIE_SIGNAL_SOURCES`. Players whose ONLY rankings come from
that set are filtered out of the top of the composite to avoid the
"draft picks who never played" pollution pattern that bit the football
repo in early PRs.
